## Overview
Committed is a daily browser game: you're shown two small real code diffs solving the same task — one written by a human, one generated by an LLM — and you pick the human. It's a Wordle-shaped puzzle about the skill the Godot project just declared impossible: distinguishing AI code from human code well enough to trust it.

## Problem
Godot banned AI-authored contributions because reviewers can't reliably tell what a person actually understands vs. what a model produced. Stroustrup's Rule says style preferences drift over time; AI has frozen a particular 'plausible' style into the water supply. Committed turns that anxiety into practice — can *you* actually spot the tells (over-commented obviousness, defensive nullchecks nobody asked for, suspiciously tidy naming, the wrong idiom for the repo)?

## How it works
Each day, one puzzle: a short problem statement ('add pagination to this endpoint') and two diffs side by side, A and B. You pick which is human. Reveal shows the answer, the source repo/PR for the human one, and the model+prompt for the AI one, plus a one-line 'tell' explainer. A streak counter and a shareable spoiler-free result grid (🟩⬛ per day) drive the loop. A weekly 'hard mode' uses AI diffs that were *actually merged* into real projects, so the tells are subtle.

## Technical approach
Static front-end (Vite + vanilla TS), diffs rendered with `diff2html`. The content pipeline is the real work: a build script pulls small, self-contained human PR diffs from permissively-licensed GitHub repos via the REST API (`/repos/{r}/pulls`, filter to <40 changed lines, single file), extracts the pre-diff file + PR title as the 'task,' then asks an LLM (Claude via the Anthropic API) to solve that same task against the same base file, producing the AI diff. Both diffs are normalized (strip trailing whitespace, unify indentation) so formatting alone isn't the tell. Data model: a JSON per day `{date, task, base_hash, diffA, diffB, human_side, source_url, model, tell}`, pre-generated in batches and committed to the repo so the game is fully static/offline. The genuinely hard part is *fair* pairs — if the AI diff is obviously worse (wrong output) the game is trivial; a filter must discard AI diffs that don't compile/pass the PR's tests, keeping only plausibly-correct impostors, ideally verified in a sandbox.

## v1 scope
- 14 hand-curated puzzles from public PRs
- side-by-side diff view, pick-the-human, reveal with source link + tell
- localStorage streak + emoji share string

## Out of scope
- automated daily generation pipeline (manual curation for v1)
- accounts, leaderboards, multiplayer
- languages beyond a couple (start with Python/JS)

## Risks & unknowns
License compliance when embedding human diffs (stick to MIT/Apache, attribute). Whether the tells generalize or just teach one model's quirks. Sandbox-verifying AI diffs compile is real infra you may punt on in v1 by hand-checking.

## Done means
A player can load today's puzzle, see two normalized diffs, pick one, get a correct/incorrect verdict with the real PR link and a tell, and copy a shareable result — with 14 days of vetted content where a knowledgeable dev scores clearly above 50% but below 90%.
