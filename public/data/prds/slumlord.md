## Overview
Slumlord is an idle/tycoon game where the currency is generated by actual malicious internet traffic. You run (or subscribe to) an SSH honeypot; every bot that logs in and runs commands is a "tenant" occupying a unit in your ramshackle building. You spend the exhaust of their attacks to add more fake services, which look more vulnerable, which attract more tenants. Sparked by HN's live SSH-honeypot viewer — turned from a security curiosity into a strangely relaxing incremental game.

## Problem
Honeypot dashboards are for analysts; they're firehoses of JSON nobody watches for fun. The internet is constantly hammering every open port with dumb bots, and that ambient hostility is oddly comforting once it's yours to observe. Nobody's made the surveillance *cozy* — an idle game is the perfect wrapper.

## How it works
Each captured session spawns a tenant sprite in a unit, tagged by behavior fingerprint (crypto-miner installer, DDoS recruiter, credential-stuffer, confused script). Tenants generate "rent" (soft currency) over time based on how many commands they ran. You spend rent to: open new fake ports/banners (more attack surface = higher spawn rate), upgrade "disrepair" (looking more exploitable boosts tenant quality), and unlock a bestiary that catalogs each malware family you've hosted. Prestige = "condemn the building" and rebuild in a spicier region. It idles: close the tab, come back to a full slum.

## Technical approach
Stack: a small Go/Node ingestor tails Cowrie honeypot JSON logs (or, for v1, replays the public honeypotlive.cc event feed) and normalizes sessions into `Tenant{fingerprint, commands[], firstSeen}`. Fingerprinting = hashing the sorted command sequence + matching against a curated ruleset (known miner/DDoS command patterns). Frontend: a canvas building grid (PixiJS) with an incremental-game economy loop. Data model persists tenants, currency, and unlocked ports in SQLite. The genuinely tricky part is behavior classification that's *entertaining but honest* — mapping messy real attack sessions onto ~12 legible "tenant species" without lying about what they did.

## v1 scope
- Ingest a local Cowrie JSON log (or replay a canned sample file)
- 4 tenant species from command-pattern rules
- One currency, one spendable upgrade (open a new port → higher spawn rate)
- Static building grid; tenants appear as sessions arrive

## Out of scope
- Actually exposing the user's own box (v1 replays logs; live honeypot is opt-in later)
- Prestige/rebirth, regions, leaderboards
- Real-time multiplayer

## Risks & unknowns
- Running a real internet-facing honeypot has security/ISP-TOS friction; must default to replay/sample data.
- Attack traffic is bursty — the game may starve or flood; needs a smoothing/backlog buffer to feel idle-paced.
- Romanticizing malware sits weird; keep it observational, never a "launch attacks" tool.

## Done means
Pointed at a sample Cowrie log, the game replays real captured sessions as tenants classified into species, accrues rent over time, lets me spend it to open a port that visibly raises the spawn rate, and keeps accruing while the tab is backgrounded — all with zero real attack surface exposed.
