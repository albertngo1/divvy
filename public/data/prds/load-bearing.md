## Overview
Load Bearing is a daily browser puzzle game about controlled collapse. Each day everyone gets the same procedurally-generated 3D structure and must identify the single load-bearing member to remove so gravity does the rest. It's Wordle-for-engineers, sparked by the Box3D physics-engine announcement.

## Problem
Most physics games are sandboxes with no goal or twitchy real-time chaos. There's no crisp, shareable, once-a-day physics puzzle with a clean 'aha' — the satisfying inverse of building: figuring out the one cut that brings it all down.

## How it works
You see a 3D assembly of beams, blocks, and joints suspended above a target zone. You may remove exactly one piece (or in harder modes, make one cut). Hit GO and the physics runs deterministically. Score = how much mass lands in the target zone / how many pieces you dislodged, vs. par. You get 3 attempts; fewer attempts and closer-to-optimal collapse = better share score (emoji grid of your collapse, Wordle-style). Because the sim is deterministic and seeded, everyone's structure and physics are identical — pure puzzle, no reflexes.

## Technical approach
Stack: TypeScript + a Box3D/Box2D-3D WASM build (or Rapier as a proven fallback) running in a Web Worker for determinism. The key requirement is a FIXED timestep with a deterministic solver and identical float behavior across machines — so all players get the same collapse from the same removal; this is the genuinely hard part (float determinism across browsers). Structures are generated by a seeded PRNG (seed = date) that stacks parametric modules and validates via a quick pre-sim that (a) the intact structure is stable and (b) at least one single removal produces a >threshold collapse — reject seeds that don't. Rendering via Three.js sharing the physics transform buffer. Backend is a static site + a tiny KV (Cloudflare Workers KV) for daily leaderboards.

## v1 scope
- One deterministic daily seed, one structure archetype
- Remove-one-piece interaction, 3 attempts
- Score vs. par + emoji share grid
- No account, localStorage streak

## Out of scope
- Cut mode, multi-piece removal
- Global leaderboard / accounts
- Level editor / user submissions
- Mobile-optimized controls

## Risks & unknowns
- Cross-browser float determinism may force lockstep or server-authoritative sim
- Generating structures with exactly one elegant solution is nontrivial
- Box3D is brand-new; API/WASM maturity unknown (Rapier fallback)

## Done means
Two different browsers loading the same day's puzzle, removing the same beam, produce pixel-comparable collapses and identical scores, and the share grid encodes the result.
